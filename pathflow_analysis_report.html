Bugs & breaking issues
Critical

Worker pipeline silently swallows errors — workspace stays "pending" forever

If any Celery task in the chain raises an exception (clone fails, embedding API rate-limits, etc.), the status column is never updated to "error". The frontend polls and waits forever. There is no on_failure handler on the chain or individual tasks.

worker/tasks/clone_task.py · pipeline.py
Critical

Workspace router imports Celery tasks at request time via sys.path hack

The POST /api/workspace handler inserts the worker directory into sys.path at runtime and then does from tasks.pipeline import run_analysis_pipeline. This will break in production (worker runs in a different container), silently fails if the path is wrong, and is not tested. The correct approach is to send a Celery task message via the broker — the worker and backend should be decoupled.

backend/app/routers/workspace.py:31–36
Critical

asyncio.create_task() inside StreamingResponse leaks — save_response never runs

The AI chat route creates a background task to save the assistant message after streaming. But asyncio.create_task() inside a generator that has already returned its StreamingResponse is not guaranteed to run — the event loop may close the scope before the task executes, and the assistant message is never saved to the DB.

backend/app/routers/ai.py:50–56
Critical

git.pr/create hardcodes "main" as PR base branch — fails for repos using "master" or custom branches

The submit PR route calls create_pull_request(..., base="main") with a hardcoded string. Most modern repos use main but many large OSS projects still use master or a different default. This will cause a GitHub API 422 error with no user-facing explanation.

The Upstash REST API is called as POST /set/{key}/{value}/EX/{ttl} with the JSON-serialised value literally in the URL path. Any cache value containing /, spaces, ?, or & will cause a malformed URL or incorrect routing. Should use the request body instead.

The route fetches the last 10 messages order_by(created_at.desc()).limit(10) then calls reversed(). The PR create route sets the remote URL to https://{user.github_token}@github.com/.... GitPython may log this URL (including the token) to stdout/stderr or in error traces. Any log aggregation service will capture the plain-text token.

The guard os.path.abspath(file_path).startswith(os.path.abspath(repo_path)) does not resolve symlinks. A malicious repo could create a symlink inside the cloned directory pointing to /etc/passwd or any other host file, and the check would pass.

Any authenticated user can hit POST /api/workspace in a loop, triggering unlimited Celery jobs and OpenRouter API calls. No per-user workspace count limit, no per-minute rate limit on AI routes. A single bad actor can exhaust free-tier quota.

Improvements & tech debt
Architecture

Decouple backend and worker properly via Celery send_task

Replace the sys.path hack in workspace router with celery_app.send_task("tasks.clone", args=[...]). The backend should only know task names (strings), not import worker code directly. This is the standard Celery pattern.

Architecture

Add Pydantic v2 schemas for all request bodies and response models

All routes use raw request.json() with no validation. Adding schemas gives free OpenAPI docs, automatic 422 errors for bad inputs, type safety, and makes the API self-documenting at /docs.

Architecture

Add Celery task failure handlers and status propagation

Wrap the pipeline with on_failure callbacks that update workspace.status = "error" and store the error message. Add a GET /api/workspace/{id}/status response that includes the error detail when status is "error".

Performance

Embed task embeds one chunk at a time — 60 files × ~3 chunks = ~180 sequential API calls

The embedding loop calls await get_embedding() one at a time with no batching beyond the inner batch_size=20 grouping. OpenRouter's embeddings endpoint supports batching multiple inputs in one call. Using batch embedding would reduce analysis time by ~10×.

worker/tasks/embed_task.py:40–44
Performance

SWR polling runs on every workspace tab — fetches file tree and workspace state simultaneously

The workspace page, IDELayout, and explore page all independently call useSWR for /workspace/{id} and /files/{id}. SWR deduplicates same-key calls within the same component tree, but the overlapping keys between parent and child components create unnecessary network requests.

Performance

Issue classifier sends all 50 issues to the LLM in one call — hits context limits on busy repos

Large repos (React, Next.js, etc.) have hundreds of issues. Truncating to 50 and sending all at once in a single prompt can hit the model's context window. Should paginate or batch in groups of 15–20 with retry logic.

shared/prompts.py:build_issue_classifier_prompt
